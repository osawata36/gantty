# 用語集

このページでは、Ganttyおよびプロジェクト管理で使用される主要な用語について解説します。

## あ行

### インデント

タスクを階層の下位レベルに移動すること。インデントされたタスクは、直前のタスクの子タスクになります。リストビューで `Tab` キーを押すか、ドラッグ&ドロップで行います。

→ 参照: [階層構造（親子関係）](/tasks/hierarchy)

### アウトデント

タスクを階層の上位レベルに移動すること。アウトデントされたタスクは、親タスクと同じ階層になります。`Shift + Tab` キーを押すか、ドラッグ&ドロップで行います。

→ 参照: [階層構造（親子関係）](/tasks/hierarchy)

## か行

### 開始日

タスクの作業を開始する予定の日付。ガントチャートビューでは、タスクバーの左端がこの日付に対応します。

→ 参照: [日程と所要日数](/tasks/dates-duration)

### ガントチャート

タスクをタイムライン上に横棒（バー）として表示するチャート。各タスクの期間、開始日、終了日を視覚的に確認でき、プロジェクト全体のスケジュールを把握するのに適しています。

20世紀初頭にヘンリー・ガントによって考案されたことからこの名前が付けられています。

→ 参照: [ガントチャートビュー](/views/gantt-view)

### カンバン

タスクをカード形式で表示し、ステータス（列）間を移動させることで進捗を管理する手法。「看板」から派生した言葉で、トヨタ生産方式から発展しました。

→ 参照: [カンバンビュー](/views/kanban-view)

### 期間

タスクの開始日から終了日までの日数。所要日数とも呼ばれます。

## さ行

### サブタスク

親タスクの下位に位置するタスク。親タスクを細分化した作業項目を表します。Ganttyでは、タスクに任意の深さのサブタスクを設定できます。

→ 参照: [階層構造（親子関係）](/tasks/hierarchy)

### 終了日

タスクの作業を完了する予定の日付。ガントチャートビューでは、タスクバーの右端がこの日付に対応します。

→ 参照: [日程と所要日数](/tasks/dates-duration)

### 進捗率

タスクがどの程度完了しているかを示すパーセンテージ（0-100%）。親タスクの進捗率は、通常、子タスクの進捗から自動計算されます。

→ 参照: [進捗管理](/tasks/progress)

### ステータス

タスクの現在の状態を示す属性。デフォルトでは「未着手」「進行中」「完了」などが用意されていますが、プロジェクトに合わせてカスタマイズできます。

→ 参照: [ステータス管理](/status/)

### 責任者

タスクの完了について最終的な責任を持つメンバー。タスクに対して1人を割り当てます。責任者は必ずしも実際の作業を行う人とは限りません（作業者は「ボール」で管理します）。

→ 参照: [タスクへの割り当て](/members/assignment)

### 所要日数

タスクを完了するために必要な日数。開始日を設定し、所要日数を入力すると、終了日が自動的に計算されます。逆に、開始日と終了日から所要日数が計算されることもあります。

→ 参照: [日程と所要日数](/tasks/dates-duration)

## た行

### タスク

プロジェクトを構成する個々の作業項目。Ganttyでは、タスクに名前、日程、ステータス、担当者などの属性を設定できます。

→ 参照: [タスクの作成・編集](/tasks/create-edit)

### タスクバー

ガントチャートビューでタスクを表す横棒。バーの長さがタスクの期間を、位置が開始日と終了日を表します。ドラッグして移動したり、端をドラッグして期間を変更したりできます。

→ 参照: [ガントチャートビュー](/views/gantt-view)

## は行

### 親タスク

1つ以上のサブタスク（子タスク）を持つタスク。親タスクは、子タスクをグループ化し、作業を論理的に整理するために使用します。

→ 参照: [階層構造（親子関係）](/tasks/hierarchy)

### ボール

タスクを「今、誰が持っているか」を示す概念。スポーツでボールを持っている選手のように、現在そのタスクに対してアクションを起こすべき人を指します。

例えば、レビュー待ちのタスクは作成者ではなくレビュアーがボールを持っている状態です。責任者とは異なり、タスクの進行状況に応じて変わることがあります。

→ 参照: [タスクへの割り当て](/members/assignment)

### プロジェクト

Ganttyで管理する作業全体のこと。1つのプロジェクトは1つの `.gantty` ファイルとして保存されます。

→ 参照: [保存と読み込み](/files/save-open)

## ま行

### マイルストーン

プロジェクトの重要な節目や目標達成日を示す特別なタスク。通常のタスクとは異なり、期間を持たず、特定の日付を示します。ガントチャートでは菱形（ひし形）で表示されることが一般的です。

マイルストーンの例：
- プロジェクト開始
- 設計完了
- リリース日
- クライアントレビュー

### メンバー

プロジェクトに参加するチームの一員。メンバーをあらかじめ登録しておくことで、タスクの責任者やボールとして割り当てることができます。

→ 参照: [メンバー登録](/members/registration)

## や行

### 優先度

タスクの重要性や緊急性を示す属性。優先度に応じてタスクの処理順序を決定するために使用します。

## ら行

### リストビュー

タスクを階層的なリスト形式で表示するビュー。WBS（作業分解構成図）を作成・編集するのに適しています。

→ 参照: [リストビュー](/views/list-view)

## わ行

### WBS（Work Breakdown Structure）

作業分解構成図。プロジェクトの成果物や作業を階層的に分解し、構造化したもの。大きな目標を達成可能な小さなタスクに分割することで、プロジェクト全体を管理しやすくします。

WBSの特徴：
- 階層構造で作業を整理
- 親タスクは子タスクの集合体
- 最下層のタスク（ワークパッケージ）は具体的な作業単位

Ganttyのリストビューは、WBSの作成と管理に最適化されています。

→ 参照: [リストビュー](/views/list-view)、[階層構造（親子関係）](/tasks/hierarchy)
