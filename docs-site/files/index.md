# ファイル操作

Ganttyでは、プロジェクトデータを`.gantty`形式のファイルとして保存・読み込みできます。このセクションでは、ファイル操作の基本と、ファイル形式について説明します。

## .gantty ファイル形式

Ganttyのプロジェクトは、`.gantty`拡張子を持つファイルに保存されます。

### ファイル形式の特徴

| 項目 | 内容 |
|-----|------|
| 拡張子 | `.gantty` |
| フォーマット | JSON形式（テキストファイル） |
| 文字コード | UTF-8 |
| バージョン | 1.0.0（現行） |

### ファイル構造

.ganttyファイルは、以下の構造を持つJSONファイルです：

```json
{
  "version": "1.0.0",
  "project": {
    "id": "プロジェクトID",
    "name": "プロジェクト名",
    "description": "プロジェクトの説明",
    "createdAt": "作成日時",
    "updatedAt": "更新日時",
    "tasks": [...],
    "resources": [...],
    "statuses": [...]
  }
}
```

### 保存されるデータ

.ganttyファイルには、以下のプロジェクトデータが保存されます：

- **プロジェクト情報**: 名前、説明、作成日時、更新日時
- **タスク**: すべてのタスクとその詳細（階層構造、日程、進捗、担当者など）
- **リソース（メンバー）**: 登録されているチームメンバー情報
- **ステータス設定**: カスタマイズしたステータスの定義

## ファイル操作の種類

Ganttyでは以下のファイル操作が可能です：

### 新規プロジェクト

新しい空のプロジェクトを作成します。デフォルトのステータス設定が適用されます。

### 保存

現在のプロジェクトをファイルに保存します。初めて保存する場合は保存先を選択、既存ファイルがある場合は上書き保存されます。

### 名前を付けて保存

現在のプロジェクトを新しいファイルとして保存します。既存のファイルを別名でコピーしたい場合に使用します。

### 開く

保存済みの.ganttyファイルを読み込み、プロジェクトを開きます。

詳しい操作方法は[保存と読み込み](./save-open)をご覧ください。

## 未保存の変更

プロジェクトに未保存の変更がある場合、以下の方法で確認できます：

- ウィンドウタイトルに `*` マークが表示されます
- 新規プロジェクト作成時や別のプロジェクトを開く前に、保存確認ダイアログが表示されます

## ファイル管理のヒント

### 命名規則

プロジェクトファイルには、内容がわかりやすい名前をつけましょう。

```
良い例:
- 2024年度_新規サービス開発.gantty
- マーケティングキャンペーン_Q1.gantty
- システムリプレイス_フェーズ1.gantty

避けるべき例:
- 無題.gantty
- project1.gantty
- test.gantty
```

### バックアップ

重要なプロジェクトは定期的にバックアップを取ることをお勧めします。「名前を付けて保存」機能を使って、日付入りのファイル名で保存するのも有効です。

```
例: プロジェクト名_2024-01-15.gantty
```

### ファイルの共有

.ganttyファイルはテキストベースのJSONファイルなので、以下の方法で共有できます：

- メール添付
- クラウドストレージ（Google Drive、Dropbox など）
- ファイル共有サービス
- バージョン管理システム（Git など）
