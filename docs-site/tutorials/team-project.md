# チームプロジェクト管理

このチュートリアルでは、Ganttyを使ってチームでプロジェクトを管理する方法を学びます。メンバーの登録からタスクへの割り当て、カンバンビューやガントチャートビューを活用したチーム運営まで、実践的なワークフローを体験しましょう。

**所要時間**: 約20分
**対象**: チームでGanttyを活用したい方
**前提**: [基本的なワークフロー](./basic-workflow)を完了していること

## シナリオ

このチュートリアルでは、以下のチーム構成で「Webサイトリニューアル」プロジェクトを管理します。

| メンバー | 役割 |
|----------|------|
| 田中太郎 | プロジェクトマネージャー |
| 鈴木花子 | デザイナー |
| 佐藤健一 | フロントエンドエンジニア |
| 山田美咲 | バックエンドエンジニア |

---

## Step 1: メンバーを登録する

プロジェクトに参加するメンバーを登録します。

### 1-1. メンバー管理画面を開く

1. サイドバーまたはメニューから「メンバー」を選択します
2. メンバー管理画面が表示されます

### 1-2. メンバーを追加する

1. 「メンバーを追加」ボタンをクリックします
2. 以下の情報を入力します:
   - **名前**: 田中太郎
   - **役割**（任意）: プロジェクトマネージャー
3. 「追加」をクリックします

同様に、残りのメンバーも追加してください。

- 鈴木花子（デザイナー）
- 佐藤健一（フロントエンドエンジニア）
- 山田美咲（バックエンドエンジニア）

::: tip メンバーの色分け
メンバーごとに色を設定すると、タスクリストやガントチャートで担当者が視覚的にわかりやすくなります。
:::

### 1-3. 登録内容を確認する

メンバー一覧に4名が表示されていることを確認します。

---

## Step 2: タスクに担当者を割り当てる

各タスクに責任者とボール（現担当者）を設定します。

### 2-1. 責任者とボールの違いを理解する

Ganttyでは、2種類の担当者設定があります。

| 種類 | 説明 | 用途 |
|------|------|------|
| **責任者** | タスクの最終責任を持つ人 | タスクの完了に責任を持つ人を明確化 |
| **ボール** | 現在タスクを担当している人 | 今誰がボールを持っているかを表現 |

::: info ボールとは
「ボールを持つ」とは、そのタスクを現在進行している人を指します。タスクの進行中にボールが移動することがあります。例: デザイナーがデザインを完成させたら、ボールをエンジニアに渡す。
:::

### 2-2. タスクに責任者を設定する

1. リストビューで「デザイン制作」タスクをクリックします
2. 詳細パネルの「責任者」ドロップダウンをクリックします
3. 「鈴木花子」を選択します

以下の担当者設定を行ってください。

| タスク | 責任者 |
|--------|--------|
| 現状分析 | 田中太郎 |
| 要件定義 | 田中太郎 |
| デザイン制作 | 鈴木花子 |
| コーディング | 佐藤健一 |
| テスト・検証 | 山田美咲 |
| 公開 | 田中太郎 |

### 2-3. ボールを設定する

プロジェクト開始時点では、最初のタスク「現状分析」のボールを設定します。

1. 「現状分析」タスクをクリックします
2. 詳細パネルの「ボール」ドロップダウンをクリックします
3. 「田中太郎」を選択します

::: tip ボールの活用シーン
- **レビュー待ち**: 作業者からレビュアーにボールを移動
- **確認待ち**: エンジニアからPMにボールを移動
- **対応依頼**: PMから担当者にボールを移動
:::

---

## Step 3: カンバンビューでステータスを管理する

カンバンビューを使って、チーム全体のタスク状況を視覚的に管理します。

### 3-1. カンバンビューに切り替える

1. ビュー切り替えで「カンバン」を選択します
2. タスクがステータス別のカラムに表示されます

### 3-2. カンバンビューの構成を理解する

カンバンビューは、ステータスごとにカラム（列）が表示されます。

| カラム | 説明 |
|--------|------|
| 未着手 | まだ開始していないタスク |
| 進行中 | 現在作業中のタスク |
| レビュー | レビュー・確認待ちのタスク |
| 完了 | 完了したタスク |

::: info ステータスのカスタマイズ
ステータスは[ステータス管理](/status/)でカスタマイズできます。チームのワークフローに合わせて追加・変更が可能です。
:::

### 3-3. ドラッグ&ドロップでステータスを変更する

1. 「現状分析」タスクのカードをドラッグします
2. 「進行中」カラムにドロップします
3. タスクのステータスが「進行中」に変わります

### 3-4. カンバンビューでの進捗確認

カンバンビューでは、以下の情報を一目で確認できます。

- 各ステータスにあるタスク数
- 誰がどのタスクを担当しているか（担当者表示）
- ボトルネックの発見（特定のカラムにタスクが溜まっている場合）

::: tip チームミーティングでの活用
カンバンビューをチームミーティングで表示し、各メンバーがボールを持っているタスクを確認しながら進捗共有を行うと効果的です。
:::

---

## Step 4: ガントチャートでスケジュールを俯瞰する

ガントチャートビューを使って、プロジェクト全体のスケジュールを把握します。

### 4-1. ガントチャートビューに切り替える

1. ビュー切り替えで「ガントチャート」を選択します
2. タスクがタイムライン上にバーで表示されます

### 4-2. スケールを調整する

プロジェクトの期間に合わせてスケールを調整します。

1. スケール切り替えボタンをクリックします
2. 目的に応じてスケールを選択します:
   - **日**: 詳細な日程確認に
   - **週**: 週単位での進捗確認に
   - **月**: プロジェクト全体の俯瞰に

### 4-3. 担当者ごとのタスクを確認する

ガントチャートでは、担当者（責任者）ごとにタスクを色分け表示できます。

1. フィルターまたは表示オプションから「担当者で色分け」を選択します
2. 各メンバーのタスクが色分けで表示されます

これにより、以下が確認できます。

- 特定のメンバーに負荷が集中していないか
- メンバー間でスケジュールの重複がないか
- 手が空いている期間があるメンバー

### 4-4. タスクバーで日程を調整する

ガントチャート上でタスクバーを操作して、日程を調整できます。

1. **移動**: タスクバーをドラッグして開始日を変更
2. **期間変更**: タスクバーの端をドラッグして期間を延長/短縮
3. **詳細確認**: タスクバーをクリックして詳細パネルを表示

::: warning タスクの依存関係
現在のバージョンでは、タスク間の依存関係（前のタスクが完了してから開始など）は自動では管理されません。日程調整時は、タスクの順序関係を意識してください。
:::

## Step 5: チーム運営のベストプラクティス

効果的なチームプロジェクト管理のためのヒントを紹介します。

### 5-1. 日次の進捗確認

毎日短い時間で進捗を確認しましょう。

1. カンバンビューで「進行中」のタスクを確認
2. ボールが誰にあるかを確認
3. ブロッカーがないかをチェック

::: tip デイリースクラム
カンバンビューを表示しながら、各メンバーが以下を報告する短いミーティングを行うと効果的です:
- 昨日やったこと
- 今日やること
- 困っていること
:::

### 5-2. 週次のスケジュール確認

週に一度、ガントチャートビューでスケジュール全体を確認しましょう。

1. 予定通り進んでいるかを確認
2. 遅延しているタスクを特定
3. 必要に応じて日程を調整

### 5-3. ボールの受け渡し

タスクのフェーズが変わったら、ボールを適切な人に渡しましょう。

| シーン | ボールの移動 |
|--------|-------------|
| デザイン完成 | デザイナー → エンジニア |
| 実装完了 | エンジニア → テスター |
| レビュー依頼 | 作業者 → レビュアー |
| 確認依頼 | 担当者 → PM |

### 5-4. ステータスの適切な更新

タスクの状態が変わったら、すぐにステータスを更新しましょう。

- 作業開始時: 未着手 → 進行中
- 作業完了時: 進行中 → 完了（またはレビュー）
- 問題発生時: → 保留（コメントで理由を記載）

## まとめ

このチュートリアルで学んだこと:

1. **メンバー登録** - プロジェクトメンバーの追加と役割設定
2. **担当者割り当て** - 責任者とボールの設定
3. **カンバンビュー活用** - ステータス管理と進捗の視覚化
4. **ガントチャート活用** - スケジュールの俯瞰と調整
5. **チーム運営** - 日次/週次確認のベストプラクティス

## チーム運営チェックリスト

- [ ] 全メンバーが登録されている
- [ ] 全タスクに責任者が設定されている
- [ ] 進行中タスクにはボールが設定されている
- [ ] ステータスが最新に更新されている
- [ ] 日程が現実的に設定されている
