# 基本的なワークフロー

このチュートリアルでは、Ganttyを使って個人プロジェクトを管理する基本的なワークフローを学びます。「Webサイトリニューアル」プロジェクトを例に、プロジェクトの作成からタスク管理、進捗追跡までを一通り体験しましょう。

**所要時間**: 約15分
**対象**: Ganttyを初めて使う方

## 事前準備

- Ganttyがインストールされていること（[インストールガイド](/getting-started/installation)を参照）
- Ganttyを起動しておくこと

---

## Step 1: プロジェクトを作成する

Ganttyを起動すると、自動的に新しいプロジェクトが作成されます。

### 1-1. プロジェクト名を設定する

1. 画面上部のプロジェクト名（デフォルトでは「新規プロジェクト」）をクリックします
2. 「Webサイトリニューアル」と入力します
3. Enterキーを押して確定します

::: tip プロジェクト名の付け方
プロジェクト名は後から変更可能です。内容がわかりやすい名前を付けましょう。
:::

### 1-2. プロジェクトを保存する

作業を始める前に、プロジェクトファイルを保存しておきましょう。

1. メニューから「ファイル」>「保存」を選択するか、`Ctrl+S`（Mac: `Cmd+S`）を押します
2. 保存先を選択し、ファイル名を入力します
3. 「保存」をクリックします

プロジェクトは `.gantty` 形式で保存されます。

---

## Step 2: タスクを追加する

プロジェクトにタスクを追加していきます。

### 2-1. 最初のタスクを追加する

1. 「タスクを追加」ボタンをクリックします
2. タスク名に「現状分析」と入力します
3. Enterキーを押して確定します

### 2-2. 複数のタスクを追加する

同様の手順で、以下のタスクを追加してください。

- 要件定義
- デザイン制作
- コーディング
- テスト・検証
- 公開

::: info 効率的なタスク追加
タスクを追加した後、すぐに次のタスク名を入力してEnterを押すと、連続してタスクを追加できます。
:::

---

## Step 3: 階層構造を作成する

タスクを親子関係で整理して、WBS（作業分解構成図）を作成します。

### 3-1. サブタスクを追加する

「現状分析」にサブタスクを追加しましょう。

1. 「現状分析」タスクを選択します
2. サブタスクを追加するアイコン（または右クリックメニューから「サブタスクを追加」）をクリックします
3. 「アクセス解析の確認」と入力してEnterを押します

同様に、以下のサブタスクも追加してください。

- 競合サイト調査
- 課題の洗い出し

### 3-2. 他のタスクにもサブタスクを追加

各親タスクに適切なサブタスクを追加しましょう。例:

**要件定義**
- 機能要件の整理
- 画面構成の検討
- スケジュール策定

**デザイン制作**
- ワイヤーフレーム作成
- ビジュアルデザイン
- レスポンシブ対応

**コーディング**
- HTMLマークアップ
- CSS実装
- JavaScript実装

**テスト・検証**
- 表示確認
- 動作テスト
- パフォーマンス確認

::: tip 階層は深くしすぎない
WBSの階層は2〜3階層程度に留めることをお勧めします。深すぎる階層は管理が複雑になります。
:::

### 3-3. ドラッグ&ドロップで階層を変更する

タスクの階層構造は、ドラッグ&ドロップで変更できます。

1. 移動したいタスクをドラッグします
2. 親にしたいタスクの上にドロップします
3. サブタスクとして配置されます

詳しくは[ドラッグ&ドロップ操作](/tasks/drag-drop)を参照してください。

---

## Step 4: 日程を設定する

各タスクに開始日と終了日を設定します。

### 4-1. タスクの詳細パネルを開く

1. 日程を設定したいタスク（例: 「現状分析」）をクリックします
2. 画面右側に詳細パネルが表示されます

### 4-2. 開始日と終了日を設定する

1. 「開始日」フィールドをクリックし、カレンダーから日付を選択します
2. 「終了日」フィールドをクリックし、カレンダーから日付を選択します

::: info 所要日数での設定
「終了日」の代わりに「所要日数」を入力することもできます。開始日と所要日数から終了日が自動計算されます。
:::

### 4-3. 全てのタスクに日程を設定

以下を目安に、各タスクに日程を設定してください。

| タスク | 所要日数の目安 |
|--------|---------------|
| 現状分析 | 5日 |
| 要件定義 | 5日 |
| デザイン制作 | 10日 |
| コーディング | 15日 |
| テスト・検証 | 5日 |
| 公開 | 1日 |

::: tip 親タスクの日程
親タスクの日程は、サブタスクの日程から自動計算されます。サブタスクの開始日〜終了日の範囲が親タスクの期間になります。
:::

### 4-4. ガントチャートビューで確認する

1. ビュー切り替えで「ガントチャート」を選択します
2. タスクがバーで表示され、日程が視覚的に確認できます
3. タスクバーをドラッグして、日程を直接調整することもできます

詳しくは[ガントチャートビュー](/views/gantt-view)を参照してください。

---

## Step 5: 進捗を記録する

プロジェクトが進行したら、進捗を記録しましょう。

### 5-1. 進捗率を設定する

1. 進捗を記録したいタスクをクリックして詳細パネルを開きます
2. 「進捗」スライダーを操作して、進捗率（0〜100%）を設定します
3. または、数値を直接入力します

### 5-2. ステータスを変更する

1. 詳細パネルの「ステータス」ドロップダウンをクリックします
2. 適切なステータスを選択します
   - 未着手
   - 進行中
   - 完了
   - 保留

::: tip 進捗率とステータスの使い分け
- **進捗率**: タスクの完了度合いを数値で表現（0〜100%）
- **ステータス**: タスクの状態を表現（未着手/進行中/完了など）

両方を併用することで、より詳細な進捗管理ができます。
:::

### 5-3. 進捗を一覧で確認する

リストビューでは、各タスクの進捗率が表示されます。プロジェクト全体の進捗状況を一目で把握できます。

---

## Step 6: プロジェクトを保存・管理する

### 6-1. 定期的に保存する

作業中は定期的に保存しましょう。

- `Ctrl+S`（Mac: `Cmd+S`）で上書き保存
- 「ファイル」>「名前を付けて保存」で別名保存

::: warning 自動保存について
現在のバージョンでは自動保存機能はありません。こまめに保存することをお勧めします。
:::

### 6-2. プロジェクトを開く

保存したプロジェクトを開くには:

1. 「ファイル」>「開く」を選択するか、`Ctrl+O`（Mac: `Cmd+O`）を押します
2. `.gantty` ファイルを選択します
3. 「開く」をクリックします

---

## まとめ

このチュートリアルで学んだこと:

1. **プロジェクトの作成と保存** - プロジェクト名の設定と.ganttyファイルでの保存
2. **タスクの追加** - 基本的なタスク追加方法
3. **階層構造の作成** - サブタスクによるWBS作成
4. **日程の設定** - 開始日・終了日・所要日数の設定
5. **進捗の記録** - 進捗率とステータスの管理
6. **ファイルの保存・管理** - 保存と読み込み
