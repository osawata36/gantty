# ステータス管理

ステータスは、タスクの進行状況を分類・管理するための機能です。各タスクには1つのステータスが割り当てられ、プロジェクト全体の状況を把握するのに役立ちます。

## デフォルトのステータス

Ganttyでは、新規プロジェクト作成時に以下の4つのステータスがあらかじめ用意されています。

| ステータス | 説明 | デフォルトカラー |
|-----------|------|-----------------|
| 未対応 | まだ着手していないタスク | グレー |
| 処理中 | 現在作業中のタスク | ブルー |
| レビュー待ち | 作業完了後、確認待ちのタスク | アンバー |
| 完了 | すべての作業が終了したタスク | グリーン |

## ステータスの役割

### タスクの分類

ステータスを設定することで、タスクを作業段階ごとに分類できます。これにより、チームメンバーがどのタスクに取り組むべきかが明確になります。

### 進捗の可視化

カンバンビューでは、ステータスごとにタスクがカラム（列）で表示されます。これにより、プロジェクト全体の進捗状況を一目で把握できます。

### 自動進捗更新

タスクのステータスを「完了」に変更すると、そのタスクの進捗率は自動的に100%に設定されます。

## ステータスの設定方法

### タスク詳細パネルから設定

1. タスクをクリックして詳細パネルを開きます
2. 「ステータス」のドロップダウンをクリックします
3. 設定したいステータスを選択します

### カンバンビューで設定

1. カンバンビューに切り替えます
2. タスクカードをドラッグします
3. 変更したいステータスのカラムにドロップします

## カスタマイズ

プロジェクトのワークフローに合わせて、ステータスを自由にカスタマイズできます。詳しくは[ステータスのカスタマイズ](./customization)をご覧ください。
