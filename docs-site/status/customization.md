# ステータスのカスタマイズ

プロジェクトのワークフローに合わせて、ステータスを自由にカスタマイズできます。ステータスの追加、名前の変更、カラーの設定、並び替え、削除が可能です。

## ステータス設定画面を開く

1. アプリケーションのメニューから「設定」を選択します
2. 「ステータス管理」をクリックします

または、カンバンビューのカラムヘッダーにある設定アイコンからもアクセスできます。

## ステータスの追加

新しいワークフローステップが必要な場合、ステータスを追加できます。

### 追加手順

1. ステータス設定画面を開きます
2. 「ステータスを追加」ボタンをクリックします
3. 以下の情報を入力します：
   - **ステータスID**: システム内部で使用される識別子（英数字）
   - **表示名**: 画面に表示される名前
   - **カラー**: ステータスを表す色
4. 「追加」ボタンをクリックします

### 追加例

```
ステータスID: testing
表示名: テスト中
カラー: #9333EA（パープル）
```

これにより、「処理中」と「レビュー待ち」の間にテスト工程を追加するなど、より詳細なワークフローを構築できます。

## ステータスの編集

既存のステータスの名前やカラーを変更できます。

### 編集手順

1. ステータス設定画面を開きます
2. 変更したいステータスの編集アイコンをクリックします
3. 表示名またはカラーを変更します
4. 「保存」ボタンをクリックします

::: tip ヒント
ステータスIDは変更できません。IDを変更する必要がある場合は、新しいステータスを作成し、古いステータスを削除してください。
:::

## カラーの設定

各ステータスには識別しやすいカラーを設定できます。

### カラー選択方法

1. ステータスの編集画面を開きます
2. カラーピッカーをクリックします
3. お好みの色を選択、またはHEXカラーコードを入力します

### 推奨カラーパターン

| ステータスタイプ | 推奨カラー | HEXコード |
|----------------|-----------|----------|
| 未着手・待機系 | グレー | `#6B7280` |
| 作業中 | ブルー | `#3B82F6` |
| 確認・レビュー系 | アンバー | `#F59E0B` |
| 完了 | グリーン | `#10B981` |
| 保留・ブロック | レッド | `#EF4444` |
| テスト系 | パープル | `#8B5CF6` |

## ステータスの並び替え

ステータスの表示順序を変更できます。この順序はカンバンビューのカラム順序にも反映されます。

### 並び替え手順

1. ステータス設定画面を開きます
2. 移動したいステータスをドラッグします
3. 目的の位置にドロップします

ステータスの順序は、プロジェクトのワークフローの流れに沿って設定することをお勧めします。

## ステータスの削除

不要になったステータスを削除できます。

### 削除手順

1. ステータス設定画面を開きます
2. 削除したいステータスの削除アイコンをクリックします
3. 確認ダイアログで「削除」を選択します

### 削除時の注意事項

::: warning 注意
- **最後の1つのステータスは削除できません** - プロジェクトには最低1つのステータスが必要です
- **既存タスクへの影響** - 削除するステータスが設定されているタスクは、自動的に残っている最初のステータスに変更されます
:::

### 削除前の確認事項

ステータスを削除する前に、以下を確認してください：

1. そのステータスが設定されているタスクがあるか
2. 設定されている場合、どのステータスに変更すべきか
3. 必要に応じて、事前にタスクのステータスを手動で変更する

## ベストプラクティス

### シンプルに保つ

ステータスが多すぎると管理が煩雑になります。5～7個程度に抑えることをお勧めします。

### 明確な命名

ステータス名は、誰が見てもタスクの状態がわかるように、具体的で明確な名前をつけましょう。

| 避けるべき例 | 推奨例 |
|-------------|-------|
| ステータス1 | 未対応 |
| 進行中2 | デザインレビュー中 |
| その他 | 保留中 |

### ワークフローの流れを反映

ステータスの順序は、実際の作業フローに沿って並べましょう。これにより、カンバンビューが直感的に使えるようになります。
