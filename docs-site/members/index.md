# メンバー管理

Ganttyでは、プロジェクトに参加するメンバーを登録し、各タスクに「責任者」と「ボール」を割り当てることができます。これにより、誰が何を担当しているのかを明確に管理できます。

## メンバー（リソース）とは

メンバーとは、プロジェクトのタスクを担当する人物のことです。Ganttyでは以下の情報を持つメンバーを登録できます。

| 項目 | 説明 |
|------|------|
| 名前 | メンバーの表示名（必須） |
| 表示色 | リストやカンバンでの識別に使用される色 |

登録したメンバーは、タスクの責任者やボール保持者として選択できるようになります。

## 責任者とボールの違い

Ganttyでは、タスクの「責任」と「実行」の所在を明確にするため、**責任者**と**ボール**という2つの属性を設けています。

### 責任者（Responsible）

**タスク完了の最終責任を負う人物**です。

- タスクの品質担保、期限管理、完了承認を行う
- タスクのライフサイクルを通じて基本的に変わらない
- プロジェクトマネージャーやチームリーダーが担当することが多い

### ボール（Ball）

**現在アクションを起こすべき人物**です。

- タスクを進めるために「今動くべき人」を示す
- 作業完了後、次の工程担当者へボールをパスする
- 「誰のところでタスクが止まっているか」を可視化する

### なぜ二元管理が必要なのか

この仕組みにより、以下のことが可能になります。

- **ボトルネックの可視化**: タスクがどこで滞留しているかが一目でわかる
- **当事者意識の喚起**: 「ボールを持っている」という状態が、行動を促す
- **責任の明確化**: 最終責任者と現担当者を区別することで、役割が明確になる

::: tip 例：レビュープロセス
1. 開発者Aがコードを書く（ボール: A、責任者: A）
2. 書き終わったらレビュアーBへボールをパス（ボール: B、責任者: A）
3. レビュー後、修正があれば開発者Aへ戻す（ボール: A、責任者: A）
4. 修正完了後、再度レビュアーBへ（ボール: B、責任者: A）
:::

## メンバー管理ダイアログの開き方

メンバーの登録・編集・削除は、メンバー管理ダイアログから行います。

### 開き方

1. 画面上部のメニューバーから「**編集**」をクリック
2. 「**メンバー管理**」を選択

または、ツールバーのメンバーアイコンをクリックしても開くことができます。

### ダイアログの構成

メンバー管理ダイアログには以下の要素があります。

- **メンバー一覧**: 登録済みのメンバーが表示されます
- **追加ボタン**: 新しいメンバーを追加します
- **編集ボタン**: 選択したメンバーの情報を編集します
- **削除ボタン**: 選択したメンバーを削除します
