# リストビュー

リストビューは、タスクを階層的なリスト形式で表示・編集するためのビューです。WBS（作業分解構成図）の作成・管理に最適で、プロジェクトの構造を整理するのに役立ちます。

## 画面レイアウト

リストビューは以下の要素で構成されています。

### メイン領域

| 領域 | 説明 |
|------|------|
| ツールバー | タスク追加ボタン、ビュー切り替えボタンなど |
| タスクリスト | 階層構造でタスクを表示するメイン領域 |
| 詳細パネル | 選択したタスクの詳細情報を表示・編集（右側） |

### タスクリストの列構成

タスクリストには以下の情報が列として表示されます。

| 列 | 説明 |
|----|------|
| タスク名 | タスクの名前（階層に応じてインデント表示） |
| 責任者 | タスクの責任者（バッジ表示） |
| ボール | 現在の担当者（バッジ表示） |
| ステータス | タスクの現在のステータス |
| 進捗 | 進捗率を示すプログレスバー |
| 開始日 | タスクの開始予定日 |
| 終了日 | タスクの終了予定日 |

## タスクの表示形式

### 階層表示（インデント）

タスクは親子関係に応じてインデント（字下げ）されて表示されます。

```
プロジェクトA
  ├─ フェーズ1
  │   ├─ タスク1-1
  │   └─ タスク1-2
  └─ フェーズ2
      ├─ タスク2-1
      └─ タスク2-2
```

インデントの深さでタスクの階層レベルを視覚的に把握できます。

### 折りたたみ/展開

親タスク（子タスクを持つタスク）の左側には、折りたたみアイコンが表示されます。

| アイコン | 状態 | 操作 |
|----------|------|------|
| ▼ | 展開中 | クリックで子タスクを折りたたむ |
| ▶ | 折りたたみ中 | クリックで子タスクを展開する |

::: tip 一括操作
- `Ctrl + クリック` (Windows) / `Cmd + クリック` (Mac): 子孫タスクも含めて全て展開/折りたたみ
:::

### 責任者・ボールバッジ

タスクに設定された責任者とボール（現担当者）は、色付きのバッジで表示されます。

- **責任者バッジ**: タスク完遂の責任を持つメンバー
- **ボールバッジ**: 現在そのタスクを実際に担当しているメンバー

バッジをクリックすると、メンバー選択のドロップダウンが表示されます。

### プログレスバー表示

進捗率は水平のプログレスバーで表示されます。

| 進捗 | 表示 |
|------|------|
| 0% | 空のバー |
| 1-99% | 進捗に応じた塗りつぶし |
| 100% | 完全に塗りつぶされたバー |

親タスクの進捗は、子タスクの進捗から自動的に計算されます（加重平均）。

## インライン編集

タスク名やその他の情報は、リスト上で直接編集できます。

### 編集の開始

タスク名または編集可能なセルを**ダブルクリック**すると、編集モードになります。

### 編集中の操作

| 操作 | 説明 |
|------|------|
| `Enter` | 編集を確定 |
| `Escape` | 編集をキャンセル（元の値に戻る） |
| `Tab` | 次の編集可能なセルへ移動 |
| `Shift + Tab` | 前の編集可能なセルへ移動 |

### 編集可能な項目

| 項目 | インライン編集 | 詳細パネル |
|------|----------------|------------|
| タスク名 | 可 | 可 |
| 責任者 | 可（ドロップダウン） | 可 |
| ボール | 可（ドロップダウン） | 可 |
| ステータス | 可（ドロップダウン） | 可 |
| 開始日 | 可（日付ピッカー） | 可 |
| 終了日 | 可（日付ピッカー） | 可 |
| 進捗 | - | 可 |
| 所要日数 | - | 可 |
| 備考 | - | 可 |

## ドラッグ&ドロップ操作

リストビューでは、ドラッグ&ドロップでタスクの並び順や階層構造を変更できます。

### 並び順の変更

タスクの行をドラッグして、別の位置にドロップすると並び順が変更されます。

1. 移動したいタスクの行をマウスでつかむ
2. 移動先の位置（他のタスクの上下）へドラッグ
3. ドロップ位置を示すインジケータが表示される
4. マウスを離すと並び順が確定

### 階層構造の変更

ドラッグ中に**左右に移動**することで、階層レベルを変更できます。

| 操作 | 結果 |
|------|------|
| 右へドラッグ | 上のタスクの子タスクになる（階層を深くする） |
| 左へドラッグ | 親タスクから独立する（階層を浅くする） |

::: warning 注意
- 親タスクを移動すると、子タスクも一緒に移動します
- 循環参照（タスクを自分自身の子孫にする）はできません
:::

### ドロップインジケータ

ドラッグ中は、ドロップ可能な位置にインジケータ（線やハイライト）が表示されます。

| インジケータ | 意味 |
|--------------|------|
| 横線 | その位置に挿入される |
| タスク全体のハイライト | そのタスクの子になる |

## タスクの選択

### 単一選択

タスクの行をクリックすると、そのタスクが選択されます。選択されたタスクは背景色がハイライトされ、右側の詳細パネルにそのタスクの詳細情報が表示されます。

### 複数選択

| 操作 | 説明 |
|------|------|
| `Ctrl + クリック` (Windows) / `Cmd + クリック` (Mac) | 個別に追加選択 |
| `Shift + クリック` | 範囲選択（最後に選択したタスクから連続して選択） |

::: info 複数選択時の操作
複数のタスクを選択している状態では、以下の一括操作が可能です。
- 一括削除
- ドラッグ&ドロップでの一括移動
:::

## キーボード操作

リストビューでは、キーボードで効率的にタスクを操作できます。

| キー | 操作 |
|------|------|
| `↑` / `↓` | 上下のタスクへ移動 |
| `Enter` | 選択中のタスクを編集開始 / 新しいタスクを追加 |
| `Delete` / `Backspace` | 選択中のタスクを削除 |
| `Tab` | 選択中のタスクを子タスクにする（インデント） |
| `Shift + Tab` | 選択中のタスクのインデントを解除 |
| `Space` | 折りたたみ/展開の切り替え |

## よくある使い方

### WBSの作成手順

1. まず大きな作業単位（フェーズや機能）を追加
2. 各作業を選択し、`Tab`キーまたは「サブタスク追加」で子タスクを作成
3. 子タスクをさらに分解して詳細化
4. 必要に応じてドラッグ&ドロップで構造を調整

### タスクの素早い入力

1. 「タスクを追加」をクリック、または最後のタスクで`Enter`
2. タスク名を入力して`Enter`
3. 続けて次のタスク名を入力
4. `Tab`で子タスクにしたり、`Shift + Tab`で階層を上げたりしながら構造化
