# カンバンビュー

カンバンビューは、タスクをステータス別のカラム（列）に分けてカード形式で表示するビューです。日々の進捗管理やチームでの作業状況の把握に最適です。

## 画面レイアウト

カンバンビューは、ステータスごとに分かれた複数のカラムで構成されています。

### カラム構成

デフォルトでは以下のステータスカラムが表示されます。

| カラム | 説明 |
|--------|------|
| 未着手 | まだ開始していないタスク |
| 進行中 | 現在作業中のタスク |
| 完了 | 完了したタスク |

::: info カスタマイズ
ステータスは設定画面でカスタマイズできます。カスタムステータスを追加すると、対応するカラムが自動的に追加されます。
詳しくは[ステータス管理](/status/)をご覧ください。
:::

### カラムヘッダー

各カラムの上部には以下の情報が表示されます。

| 要素 | 説明 |
|------|------|
| ステータス名 | カラムのステータス（例：進行中） |
| タスク数 | そのステータスのタスク数（例：5件） |
| カラーインジケータ | ステータスに設定された色 |

## タスクカード

各タスクはカード形式で表示され、以下の情報が含まれます。

### カードの構成要素

| 要素 | 説明 | 位置 |
|------|------|------|
| タスク名 | タスクの名前 | カード上部 |
| 親タスク名 | 階層構造での親タスク名（薄い文字） | タスク名の上 |
| 責任者バッジ | 責任者のアイコンまたは名前 | カード右上 |
| ボールバッジ | 現担当者のアイコンまたは名前 | カード右上 |
| 進捗バー | 進捗率を示すプログレスバー | カード下部 |
| 期限 | 終了日（締め切り日） | カード下部 |

### カードの色分け

カードの左端には、ステータスに応じた色のラインが表示されます。また、以下の条件で視覚的な強調が行われます。

| 条件 | 表示 |
|------|------|
| 期限超過 | カードの背景が薄い赤色に |
| 期限間近（3日以内） | 期限表示が警告色（オレンジ）に |
| 高進捗（80%以上） | 進捗バーが緑色に |

## 末端タスクのみ表示される理由

カンバンビューでは、**末端タスク（リーフタスク）のみ**が表示されます。

### 末端タスクとは

末端タスクとは、子タスクを持たない最下層のタスクのことです。

```
プロジェクト（非表示）
  └─ フェーズ1（非表示）
       ├─ タスクA ← 表示される（末端）
       └─ タスクB ← 表示される（末端）
```

### なぜ末端タスクのみなのか

| 理由 | 説明 |
|------|------|
| 実際の作業単位 | 末端タスクが実際に手を動かす作業単位だから |
| 二重計上の防止 | 親タスクと子タスクの両方を表示すると、作業が重複してカウントされてしまう |
| カンバンの本質 | カンバンは「作業の流れ」を管理するもの。実際に流れるのは末端タスク |
| 見やすさ | 親タスクを除外することで、実際の作業に集中できる |

::: tip 親タスクを確認したいとき
カードに表示される親タスク名をクリックすると、リストビューでその親タスクを表示できます。または、リストビューに切り替えて階層構造を確認してください。
:::

## ドラッグ&ドロップ操作

カンバンビューの主要な操作は、カードのドラッグ&ドロップです。

### ステータスの変更

タスクカードを別のカラムにドラッグ&ドロップすることで、ステータスを変更できます。

1. 移動したいタスクカードをマウスでつかむ
2. 目的のステータスカラムへドラッグ
3. カラム内の希望の位置でドロップ
4. タスクのステータスが自動的に変更される

::: info 確認ダイアログ
「完了」ステータスへの変更時、進捗率が100%でない場合は確認ダイアログが表示されることがあります。
:::

### カラム内での並び替え

同じカラム内でカードをドラッグ&ドロップすると、表示順序を変更できます。

- 上にドラッグ：優先度を上げる（上位に表示）
- 下にドラッグ：優先度を下げる（下位に表示）

### ドロップインジケータ

ドラッグ中は、ドロップ可能な位置にインジケータが表示されます。

| インジケータ | 意味 |
|--------------|------|
| 横線 | その位置に挿入される |
| カラムのハイライト | そのカラム（ステータス）に移動 |

## タスクの選択と詳細表示

### タスクの選択

タスクカードをクリックすると、そのタスクが選択され、詳細パネルが表示されます。

### 詳細パネルでの編集

選択したタスクの詳細は、右側の詳細パネルで編集できます。

- タスク名の変更
- 責任者・ボールの割り当て
- 開始日・終了日の設定
- 進捗率の更新
- ステータスの変更（ドロップダウン）
- 備考の追加

## フィルタリング

カンバンビューでは、表示するタスクを絞り込むことができます。

### 担当者でフィルタ

ツールバーの担当者フィルタを使用して、特定のメンバーが関わるタスクのみを表示できます。

| フィルタ | 表示されるタスク |
|----------|------------------|
| すべて | すべてのタスク |
| 責任者で絞り込み | 選択したメンバーが責任者のタスク |
| ボールで絞り込み | 選択したメンバーがボールのタスク |
| 自分のタスク | 自分が責任者またはボールのタスク |

### 期限でフィルタ

期限フィルタを使用して、期限に関連するタスクを絞り込めます。

| フィルタ | 表示されるタスク |
|----------|------------------|
| すべて | すべてのタスク |
| 今日期限 | 今日が終了日のタスク |
| 今週期限 | 今週中が終了日のタスク |
| 期限超過 | 終了日を過ぎたが完了していないタスク |

## カラムの操作

### カラムの折りたたみ

カラムヘッダーをクリックすると、そのカラムを折りたたんで幅を狭くできます。完了タスクなど、普段見ない列を折りたたんでおくと、画面を有効活用できます。

### カラムの並び替え

カラムの表示順序は、ステータス管理画面で変更できます。ワークフローに合わせて、左から右へ作業が流れるように配置することをお勧めします。

## キーボード操作

| キー | 操作 |
|------|------|
| `←` / `→` | 選択中のタスクを前後のカラムへ移動 |
| `↑` / `↓` | 同じカラム内で上下のタスクを選択 |
| `Enter` | 選択中のタスクの詳細を開く |
| `Space` | 選択中のタスクを次のステータスへ移動 |

## よくある使い方

### 毎日の進捗確認

1. カンバンビューを開く
2. 「進行中」カラムを確認し、作業中のタスクを把握
3. 完了したタスクを「完了」カラムへドラッグ
4. 次に着手するタスクを「未着手」から「進行中」へドラッグ

### チーム進捗ミーティング

1. カンバンビューを画面共有
2. 各カラムのタスク数で全体進捗を確認
3. 「進行中」のタスクについて、担当者から状況報告
4. ブロッカー（障害）があれば議論
5. 必要に応じてその場でステータスを更新

### ボトルネックの発見

1. 各カラムのタスク数を比較
2. 特定のカラムにタスクが溜まっていないか確認
3. 溜まっているカラムがあれば、その原因を調査
   - 例：「レビュー待ち」に溜まっている → レビュー体制の見直し

### WIP制限（作業中タスク数の制限）

効率的な作業のため、「進行中」のタスク数を制限することを推奨します。

| チーム規模 | 推奨WIP数 |
|------------|-----------|
| 1人 | 1〜2件 |
| 小チーム（3〜5人） | 3〜5件 |
| 大チーム（6人以上） | チーム人数程度 |

::: warning 注意
「進行中」のタスクが多すぎると、コンテキストスイッチによる効率低下や、完了までの時間が長くなる傾向があります。
:::
