# Step 8: ネットワーク図

## 目標

- ネットワーク図の見方を理解する
- WBSモードと依存モードを使い分ける
- 依存関係を確認・編集する
- クリティカルパスを把握する

## ネットワークビューに切り替える

ツールバーの「ネットワーク」ボタンをクリックします。

![ネットワーク図](/images/07_network_view.png)

## 2つの表示モード

ネットワークビューには2つのモードがあります。

### WBSモード

タスクの親子関係をマインドマップ形式で表示します。

- 階層構造を視覚的に把握
- サブタスクの追加・削除
- ドラッグで階層変更

### 依存モード

タスク間の依存関係（FS、SS、FF、SF）を表示します。

- 依存関係の確認・編集
- クリティカルパスの表示
- 所要日数の表示

## 画面構成

### ノード（タスク）

各タスクが四角形のノードとして表示されます。

- **タスク名**: ノード内に表示
- **所要日数**: タスク名の下に表示
- **色**: ステータスに応じた色

### エッジ（依存関係）

タスク間の依存関係が矢印で表示されます。

- **矢印の方向**: 前タスク → 後タスク
- **ラベル**: 依存関係の種類（FS、SS、FF、SF）

## 依存関係の編集

### 新しい依存関係を作成

1. 開始タスクのノードにマウスを合わせる
2. 右端に表示される接続ハンドル（オレンジ色）をドラッグ
3. 終了タスクにドロップ

### 依存関係を削除

1. エッジ（矢印）をクリックして選択
2. Delete キーを押す

## ズーム操作

- **ズームイン**: より詳細に表示
- **ズームアウト**: より広い範囲を表示
- **リセット**: 100%に戻す

## タッチ/スタイラス操作（iPad対応）

iPadやタブレットでは以下のタッチ操作が使えます：

| 操作 | 方法 |
|------|------|
| 選択 | タップ |
| 名前編集 | ダブルタップ |
| 階層変更（WBSモード） | 長押し → ドラッグ |
| サブタスク追加（WBSモード） | 選択後に「+」をタップ |

## 自動レイアウト

「自動レイアウト」ボタンをクリックすると、依存関係に基づいてノードが自動配置されます。

## クリティカルパス

::: info クリティカルパスとは
プロジェクト全体の所要期間を決定する最長の作業経路です。クリティカルパス上のタスクが遅れると、プロジェクト全体が遅れます。

クリティカルパス上のタスクは赤く強調表示されます。
:::

## 演習

1. WBSモードでタスクの階層構造を確認してみましょう
2. 依存モードに切り替えて依存関係を確認してみましょう
3. 新しい依存関係を追加してみましょう
4. 不要な依存関係を削除してみましょう
