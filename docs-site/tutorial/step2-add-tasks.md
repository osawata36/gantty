# Step 2: タスクを追加する

## 目標

- タスクを追加する
- タスク名を編集する
- タスクを削除する

## シナリオ

Webサイトリニューアルプロジェクトの大まかな作業を洗い出します。

## タスクを追加する

1. 「タスクを追加」ボタンをクリック
2. タスク名を入力（例：「企画・設計フェーズ」）
3. Enter キーを押して確定

以下のタスクを追加してみましょう：

- 企画・設計フェーズ
- デザイン制作
- 開発・実装
- テスト・検証
- 本番リリース

![タスクを追加した画面](/images/02_list_view_with_tasks.png)

## タスク名を編集する

タスク名を間違えた場合は、以下の方法で修正できます：

1. タスク名をダブルクリック
2. 新しい名前を入力
3. Enter キーで確定（または Escape でキャンセル）

## タスクを削除する

不要なタスクは削除できます：

1. タスクにマウスを合わせる
2. 右側に表示されるゴミ箱アイコンをクリック

::: warning
タスクを削除すると、そのサブタスクも一緒に削除されます。
:::

## ポイント

::: tip WBS（Work Breakdown Structure）の考え方
プロジェクトは大きな作業から始めて、徐々に細かく分解していきます。
- 最初は大まかなフェーズを洗い出す
- 各フェーズを具体的な作業に分解する（次のステップで行います）
:::
