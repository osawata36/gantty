# 進捗管理

タスクの進捗率を設定して、作業の進み具合を管理する方法を説明します。進捗を適切に記録することで、プロジェクト全体の状況を把握できます。

## 進捗率とは

進捗率は、タスクの完了度を0%〜100%の数値で表したものです。

| 進捗率 | 意味 |
|--------|------|
| 0% | 未着手（まだ作業を開始していない） |
| 1〜99% | 作業中（一部完了） |
| 100% | 完了（すべての作業が終了） |

::: tip ポイント
進捗率は「どの程度作業が終わったか」を表す指標です。主観的な判断でも構いませんが、できるだけ客観的な基準（成果物の完成度、チェックリストの消化率など）で設定すると、正確な状況把握ができます。
:::

## 進捗率の設定方法

進捗率を設定する方法はいくつかあります。

### 詳細パネルで設定

1. 進捗を設定したいタスクをクリックして選択します
2. 画面右側に詳細パネルが表示されます
3. 「**進捗率**」のスライダーまたは数値入力欄で値を設定します

**スライダー：**
- スライダーをドラッグして進捗率を調整します
- 視覚的に設定できるので、おおよその進捗を素早く入力できます

**数値入力：**
- 数値入力欄に直接数値を入力します
- 0〜100の整数を入力してください

### リストビューでのインライン編集

1. リストビューで進捗率の列を探します
2. 進捗率の値をクリックします
3. 入力欄が表示されるので、値を入力します
4. **Enter** キーを押すか、入力欄の外をクリックして確定します

### よく使う進捗率

| 値 | 使用場面 |
|----|---------|
| 0% | タスクを作成した直後、未着手状態 |
| 25% | 作業に着手し、序盤の作業が完了 |
| 50% | 作業の半分が完了 |
| 75% | 作業の大部分が完了、仕上げの段階 |
| 100% | すべての作業が完了 |

::: info 補足
10%刻みや25%刻みなど、キリの良い数値で管理すると、チーム内で認識を合わせやすくなります。
:::

## 親タスクの進捗自動計算

子タスク（サブタスク）を持つ親タスクの進捗率は、子タスクの進捗率から自動的に計算されます。

### 計算方法

親タスクの進捗率は、子タスクの進捗率を**所要日数で重み付けした加重平均**で計算されます。

**計算式：**
```
親タスクの進捗率 = Σ(子タスクの進捗率 × 子タスクの所要日数) / Σ(子タスクの所要日数)
```

### 計算例

```
親タスク: Webサイト制作
├── デザイン作成    所要日数: 5日  進捗: 100%
├── コーディング    所要日数: 10日  進捗: 50%
└── テスト         所要日数: 5日  進捗: 0%

計算:
(100% × 5日 + 50% × 10日 + 0% × 5日) / (5日 + 10日 + 5日)
= (500 + 500 + 0) / 20
= 50%

→ 親タスクの進捗率: 50%
```

### 所要日数が未設定の場合

子タスクに所要日数が設定されていない場合は、**単純平均**で計算されます。

```
親タスク: 企画
├── アイデア出し    進捗: 100%
├── 検討会議       進捗: 100%
└── 企画書作成     進捗: 50%

計算:
(100% + 100% + 50%) / 3 = 83.3%

→ 親タスクの進捗率: 83%（小数点以下切り捨て）
```

::: warning 注意
親タスクの進捗率は自動計算されるため、直接編集することはできません。進捗を変更するには、子タスクの進捗率を変更してください。
:::

## 進捗率とステータスの関係

進捗率とステータスは関連していますが、別々に管理されます。

| ステータス | 一般的な進捗率 |
|-----------|--------------|
| 未着手 | 0% |
| 進行中 | 1〜99% |
| 完了 | 100% |

::: tip ポイント
ステータスを「完了」に変更すると、進捗率を自動的に100%に設定するオプションがあります。設定から変更できます。
:::

### ステータス変更時の進捗率連動

デフォルトでは、ステータスを変更しても進捗率は自動変更されません。ただし、以下の設定を有効にすることで連動させることができます。

| 設定 | 動作 |
|------|------|
| ステータス「完了」で進捗100%にする | 「完了」に変更すると進捗率が自動的に100%になる |
| 進捗100%でステータス「完了」にする | 進捗率を100%にするとステータスが自動的に「完了」になる |

## 進捗の表示

進捗率は様々な場所で確認できます。

### リストビュー

- 進捗率の列に数値が表示されます
- 進捗バー（オプション）で視覚的に確認することもできます

### ガントチャートビュー

- タスクバーの中に進捗バーが表示されます
- 進捗率に応じてタスクバーの一部が塗りつぶされます

**例：進捗50%のタスクバー**
```
[██████████░░░░░░░░░░] 50%
```

### カンバンビュー

- タスクカードに進捗率が表示されます
- 進捗バー（小）で視覚的に確認できます

## 進捗管理のベストプラクティス

効果的に進捗を管理するためのヒントです。

### 定期的に更新する

- 毎日の終わりに進捗を更新する習慣をつけましょう
- 週次のレビューで全タスクの進捗を見直すのも効果的です

### 客観的な基準を設ける

主観的な「だいたい半分」ではなく、客観的な基準で進捗率を決めましょう。

**例：ドキュメント作成タスクの場合**
| 進捗率 | 基準 |
|--------|------|
| 0% | 未着手 |
| 25% | アウトライン完成 |
| 50% | 初稿完成 |
| 75% | レビュー完了、修正中 |
| 100% | 最終版完成、提出済み |

**例：開発タスクの場合**
| 進捗率 | 基準 |
|--------|------|
| 0% | 未着手 |
| 20% | 設計完了 |
| 60% | 実装完了 |
| 80% | テスト完了 |
| 100% | レビュー完了、マージ済み |

### 正確さより継続性を重視

- 細かい進捗率（63%など）にこだわりすぎない
- 10%刻みや25%刻みなど、ざっくりした値でも継続して記録することが重要

### チームで認識を合わせる

- 進捗率の基準をチームで共有しましょう
- 「50%の定義」がメンバー間で異なると、正確な状況把握ができません

## よくある質問

### Q: 進捗率は手入力するしかないですか？

A: 現在のバージョンでは手動入力が基本です。ただし、子タスクを持つ親タスクは自動計算されるため、最下層のタスクのみ手動入力すれば、上位のタスクは自動的に計算されます。

### Q: 進捗率0%のタスクは表示を隠せますか？

A: フィルター機能を使って、特定の進捗率のタスクのみを表示することができます。リストビューの上部にあるフィルターオプションを使用してください。

### Q: 進捗が90%のまま長期間停滞しています

A: いわゆる「90%症候群」です。残り10%の作業内容を具体的に洗い出し、サブタスクに分解することをお勧めします。これにより、残作業が明確になり、進捗が進みやすくなります。
